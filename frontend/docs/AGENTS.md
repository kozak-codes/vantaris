# AGENTS.md — Vantaris Project Guide

## Project Overview

Vantaris is a browser-based 3D hex-globe RTS game built with TypeScript and Three.js, bundled with Vite. Phase 1 delivers the interactive globe, hexagonal grid, fog-of-war system, and terrain biomes.

## Architecture

```
src/
├── main.ts              Entry point — wires rendering, systems, camera, HUD, debug API
├── constants.ts         All tunable constants (subdivision level, biome weights, fog, camera)
├── types/
│   └── index.ts          Shared interfaces & enums (HexCell, BiomeType, FogState, configs)
├── globe/
│   ├── HexGrid.ts        Geodesic hex grid: icosahedron subdivision → dual graph
│   ├── GlobeRenderer.ts  Three.js scene: cell meshes, borders, atmosphere glow, starfield
│   └── terrain.ts        Seeded biome assignment with weighted random distribution
├── systems/
│   └── FogOfWar.ts       Fog state machine: Unexplored → Explored → Visible
├── camera/
│   └── CameraControls.ts Pointer/touch drag, zoom, momentum inertia
├── debug/
│   └── DebugAPI.ts       Exposed global `window.vantaris` debug interface
└── ui/
    └── HUD.ts            HTML/CSS overlays — legend, tooltip, wordmark
```

### Key Design Decisions

- **Decoupled rendering**: `FogOfWar` manages cell state; `GlobeRenderer` reads state and applies colors. No direct mesh manipulation from systems.
- **Dual graph geometry**: The hex grid is generated by constructing a dual graph of a subdivided icosahedron. Each primal vertex becomes a cell center; each primal face centroid becomes a cell vertex. This produces ~12 pentagons (at icosahedron vertices) and ~630 hexagons.
- **Cell winding**: Dual vertices are sorted radially (by angle in tangent space) around each cell center to ensure outward-facing triangle winding. The fan triangulation uses `(center, v[i], v[(i+1) % n])` to close the polygon correctly for both pentagons (5 vertices) and hexagons (6 vertices).
- **DoubleSide material**: Cell meshes use `THREE.DoubleSide` for robustness against any residual winding inconsistencies.
- **Pivot-based rotation**: The globe is placed inside a `THREE.Group` (pivot). The camera stays fixed; the pivot rotates using premultiplied quaternions for world-space Y and X rotations. This avoids the "upside-down jank" problem where local-axis rotation would invert controls.
- **Initial camera alignment**: After `revealStartingTerritory()`, the pivot is rotated so the revealed cluster faces the camera direction `(0, 0, 1)`.

## Development Workflow

### Starting the dev server

```bash
npm install
npm run dev
```

Vite opens the browser automatically. Files are watched — any save triggers hot module replacement.

### Checking for compilation errors

Vite displays errors as an overlay in the browser. For a terminal-level type check:

```bash
npx tsc --noEmit
```

For a production build (also catches issues):

```bash
npm run build
```

### Using Chrome DevTools with this project

The opencode agent has Chrome DevTools integration. Key workflow:

1. **Start Vite** (`npm run dev`) — the dev server runs in the background
2. **Navigate** the browser to `http://localhost:5173`
3. **Take snapshots** (`chrome-devtools_take_snapshot`) to inspect the DOM/a11y tree — shows HUD elements, tooltips, state
4. **Evaluate scripts** (`chrome-devtools_evaluate_script`) to inspect runtime state, verify cell counts, check fog states, debug geometry
5. **Check console** (`chrome-devtools_list_console_messages`) for errors and logs after code changes
6. **Reload** (`chrome-devtools_navigate_page` with `type: "reload"`) after making source changes — Vite HMR handles most updates automatically, but a full reload ensures clean state
7. **Take screenshots** (`chrome-devtools_take_screenshot`) to visually confirm rendering

### Debugging tips

- Add `(window as any).__variable = value` in source to expose state for DevTools inspection
- Use `console.log` statements — they appear in `chrome-devtools_list_console_messages`
- Check `npx tsc --noEmit` for type errors before relying on Vite overlay alone
- The 3D canvas is opaque; use `evaluate_script` to query Three.js scene state (mesh counts, positions, etc.)

## Debug API

The game exposes a `window.vantaris` object in the browser console for live debugging. This is available in dev mode and is the primary way to inspect and manipulate game state via Chrome DevTools (`chrome-devtools_evaluate_script`).

### Fog Controls

```js
vantaris.fog.revealAll()        // Set all cells to Visible — see the full globe
vantaris.fog.hideAll()          // Set all cells to Unexplored — full fog
vantaris.fog.revealCell(id)    // Expand from a Visible cell (same as clicking it)
vantaris.fog.revealTerritory() // Re-roll starting territory (random cluster of ~7)
vantaris.fog.revealTerritory(id) // Reveal a territory starting from a specific cell
vantaris.fog.getState(id)      // Get fog state string for a cell: "Visible" | "Explored" | "Unexplored"
vantaris.fog.count()            // Returns { visible, explored, unexplored } counts
```

### Camera Controls

```js
vantaris.camera.focusCell(id)  // Rotate globe to face a specific cell
vantaris.camera.zoom(distance) // Set zoom distance (clamped 7–25)
vantaris.camera.resetRotation() // Reset globe rotation to identity
vantaris.camera.getZoom()      // Get current camera distance
```

### State Inspection

```js
vantaris.state.cell(id)    // Full cell info: { id, biome, fog, isPentagon, center, neighborIds, neighborCount }
vantaris.state.gridInfo    // Grid summary: { total, pentagons, hexagons, vertices }
vantaris.state.fps()       // Current frames per second
```

### Direct Access

```js
vantaris.grid            // The raw HexGridData: { cells, vertices, adjacency }
vantaris.fogSystem       // The FogOfWar instance
vantaris.renderer        // The GlobeRenderer instance
vantaris.cameraControls // The CameraControls instance
vantaris.pivot           // The THREE.Group pivot the globe rotates in
```

### Typical Debug Workflows

**See the whole globe at once:**
```js
vantaris.fog.revealAll()
```

**Reset fog and start fresh:**
```js
vantaris.fog.hideAll()
vantaris.fog.revealTerritory()
vantaris.camera.focusCell(vantaris.fog.count().visible) // rough center
```

**Inspect a cell and its neighbors:**
```js
vantaris.state.cell(42)
// { id: 42, biome: "Forest", fog: "Visible", isPentagon: false, center: [...], neighborIds: [37, 38, 43, ...], neighborCount: 6 }
```

**Zoom in and focus on a specific area:**
```js
vantaris.camera.focusCell(100)
vantaris.camera.zoom(9)  // Closer than default 14.2
```

### Adding New Debug Commands

To add a new debug command:

1. Open `src/debug/DebugAPI.ts`
2. Add the method to the appropriate namespace (`fog`, `camera`, `state`) or create a new one
3. Add the method to the `DebugAPI` interface at the top of the file
4. If it's a new namespace, also add it to the console.log help text in `src/main.ts`
5. Update this documentation section

The DebugAPI pattern is:
- Commands that mutate state **should call the existing system methods** (e.g., `fogOfWar.expandFromCell()`, `globeRenderer.forceColorUpdate()`) rather than directly mutating data
- Commands that read state **should return plain objects** (not class instances) so they serialize cleanly in the console
- All commands **should log to console** with a `[debug]` prefix for discoverability

### HexGrid
- `generateHexGrid(): HexGridData` — returns `{ cells, vertices, adjacency }`

### GlobeRenderer
- `constructor(parent: Object3D, grid: HexGridData, scene?: Scene)` — parent is the pivot group; scene for starfield
- `forceColorUpdate()` — immediately sets all cell mesh colors to their current fog state (call after bulk fog state changes)
- `updateFogColors(deltaMs: number)` — lerps cell mesh colors toward fog-correct target
- `beginRevealAnimation(cellId: number, fromFog: FogState)` — starts opacity transition on a cell
- `getCellAtIntersection(intersection): number | null` — returns cell ID from a raycast intersection
- `getGlobeGroup(): Group` — returns the group containing cell meshes (useful for raycasting)
- `updateGlow(camera: Camera)` — updates atmosphere glow shader uniforms

### FogOfWar
- `revealStartingTerritory(): number[]` — randomly reveals ~7-cell cluster, returns revealed cell IDs
- `getStartingCenter(): number[] | null` — returns the average center position of the starting cluster
- `expandFromCell(cellId: number): number[]` — reveals neighbors of a visible cell, returns newly visible IDs
- `getCellInfo(cellId: number): { biome, fog, isPentagon } | null` — returns display info for a cell

### CameraControls
- `constructor(camera, canvas, pivot)` — binds pointer/touch/wheel events
- `update()` — call each frame; applies inertia, smooth zoom
- `getPivot(): Group` — returns the pivot group

### HUD
- `constructor()` — builds legend, caches DOM elements
- `showTooltip(cellId, biome, fog, isPentagon)` — updates and shows the info tooltip
- `hideTooltip()` — hides the tooltip

## Constants Reference

All tunable values live in `src/constants.ts`:

| Constant | Purpose | Default |
|----------|---------|---------|
| `GLOBE_CONFIG.radius` | Sphere radius | 5 |
| `GLOBE_CONFIG.subdivideLevel` | Icosahedron subdivision (2=~162 cells, 3=~642) | 3 |
| `GLOBE_CONFIG.borderWidth` | Border line offset above surface | 0.3 |
| `BIOME_CONFIGS` | Array of `{ type, color, weight }` per biome | See file |
| `FOG_CONFIG.unexploredColor` | Color for unexplored cells | `#0a0a0a` |
| `FOG_CONFIG.exploredSaturation` | HSL saturation fraction for explored cells | 0.25 |
| `FOG_CONFIG.exploredBrightness` | HSL brightness fraction for explored cells | 0.35 |
| `FOG_CONFIG.revealedCellCount` | Starting territory cluster size | 7 |
| `FOG_CONFIG.revealAnimationMs` | Fog reveal transition duration | 300 |
| `CAMERA_CONFIG.minDistance` | Minimum zoom distance | 7 |
| `CAMERA_CONFIG.maxDistance` | Maximum zoom distance | 25 |
| `CAMERA_CONFIG.rotationDamping` | Inertia decay factor (0-1) | 0.92 |
| `CAMERA_CONFIG.zoomSpeed` | Zoom sensitivity multiplier | 1.0 |
